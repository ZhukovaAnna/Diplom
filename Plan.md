## Раздел 2. Перечень используемых инструментов.
Среда разработки - IntelliJ IDEA CE.
Благадаря встроенному редактору кода позволяет сократить число ошибок и минимизировать время разработки.

Язык разработки - Java 11.
Java распространненая среда разработки, а также универсален. Java позволяет подключать к проекту различные библиотеки, обеспечивающие гибкую реализацию потребностей проекта.

Система контроля версий - Git.
Система контроля версий представляет собой программное обеспечение, которое позволяет отслеживать изменения в документах, при необходимости производить их откат, определять, кто и когда внес исправления.

Тестовый фреймворк - JUNIT 5 (TestNG)
Распространенный тестовый фреймворк со множеством аннотаций.

Selenium Web Driver с оберткой Selenide
Для тестирования через веб-интерфейс используем Selenium Web Driver. 

Lombok
Аннотации помогут нам сократить размер кода и улучшить читаемость.

СУБД MySQL (PostgreSQL)
Эти СУБД поддерживаются приложением (как заявлено).

Node.js(Контейнере Docker)
Данное приложение понадобится для запуска симулятора банковских серверов.

### Раздел 3. Перечень возможных рисков при автоматизации.

Увеличение времени и стоимиости на тестирование (тестирование некоторых полей быстрее и дешевле провести ручным тестированием). 
При ошибках в коде, возможно ложное срабатывание тестов. Изменение кода в  production- версии влечет изменение кода тестов, следовательно, увеличение времени и стоимости тестирования.
